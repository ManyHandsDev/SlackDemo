# Slack Clone

A full-featured Slack replica with real-time messaging, voice/video calls, huddles, collaborative canvas editing, workflows, and 30+ features — built with Next.js 14, Prisma, Socket.IO, and WebRTC.

## Features

### Core Messaging
- **Rich text editor** — Tiptap-based editor with mentions, links, code blocks (syntax highlighted), emoji, tables, task lists, and slash commands
- **Threads** — Reply threads with reply count tracking
- **Reactions** — Emoji reactions with celebration confetti animations
- **Pins & Bookmarks** — Pin messages to channels, bookmark messages personally
- **Search** — Full-text search across messages, people, and files
- **Scheduled Messages** — Schedule messages for future delivery (cron job runs every 60s)
- **Link Previews** — Automatic OG metadata scraping with SSRF protection and 7-day cache
- **GIF Search** — Tenor API v2 integration (optional)
- **Custom Emoji** — Upload workspace-specific custom emoji
- **Typing Indicators** — Real-time per-channel typing state

### Channels & Workspaces
- **Channel types** — Public, Private, DM, and Group DM (up to 9 members)
- **Multi-workspace** — Owner/Admin/Member roles with workspace switching
- **Channel categories** — Per-user drag-and-drop channel organization
- **Archive support** — Archive and restore channels
- **Unread tracking** — Per-channel unread counts with mention detection
- **Read receipts** — Per-channel read state broadcasting

### Voice & Video
- **1:1 Calls** — Voice and video calls via WebRTC (simple-peer) with call signaling over Socket.IO
- **Huddles** — Persistent group audio/video channels (Slack-style huddles) with media toggle
- **Screen sharing** — Supported through WebRTC media streams
- **TURN server support** — Optional coturn configuration for cross-network calls

### Collaboration
- **Canvas** — Collaborative document editing via Tiptap + Yjs with version history
- **Polls** — In-message polls with multi-choice, timed expiry, and real-time vote updates
- **Workflows** — Automation engine with triggers (message posted, member joined, reaction added, scheduled) and actions (post message, send notification, add reaction, assign role)

### Platform
- **File uploads** — Local filesystem (dev) or S3 (production), 10MB limit, image thumbnails via sharp
- **Notifications** — Real-time notifications for mentions, DMs, thread replies, and reactions
- **Presence** — Online/offline/away status with heartbeat system (30s heartbeat, 90s timeout)
- **Admin panel** — Workspace administration with audit logging
- **Themes** — Dark/light/system theme support
- **Keyboard shortcuts** — Customizable keyboard shortcuts
- **Virtualized lists** — react-virtuoso for performant message rendering
- **Animations** — Framer Motion transitions throughout

## Prerequisites

- **Node.js 18+**
- **pnpm** (preferred) or npm

That's it for development. The app uses embedded SQLite by default — no database installation needed.

### For Production

- **PostgreSQL** database
- **AWS S3** bucket for file storage
- **coturn** TURN server for cross-network WebRTC calls
- **TLS certificates** for HTTPS (required for WebRTC `mediaDevices` on non-localhost)

| Variable | Purpose | Default |
|----------|---------|---------|

| `AUTH_SECRET` | NextAuth JWT encryption secret | Auto-generated by `startup.sh` |

| Variable | Purpose | Get a key at |
|----------|---------|-------------|

### WebRTC / Calls (optional)

| Variable | Purpose |
|----------|---------|
| `NEXT_PUBLIC_TURN_URL` | TURN server URL (e.g., `turn:your-server:3478`) |
| `NEXT_PUBLIC_TURN_USERNAME` | TURN server username |
| `NEXT_PUBLIC_TURN_CREDENTIAL` | TURN server password |

Calls work on localhost without a TURN server (uses Google STUN servers). For cross-network calls, you need a TURN server — see the coturn setup instructions in `startup.sh`.

### App Configuration

| Variable | Purpose | Default |

## Setup

### 1. Clone and install

```bash
git clone https://github.com/ManyHandsDev/SlackDemo.git
cd SlackDemo
pnpm install   # or npm install
```

### 2. Configure environment

```bash
cp .env.example .env
```

Edit `.env` — at minimum, set `AUTH_SECRET`:

```bash
# Generate a secret
openssl rand -hex 32
```

### 3. Set up the database

```bash
pnpm db:generate   # Generate Prisma client
pnpm db:push       # Create database schema
pnpm db:seed       # Seed demo data
```

### 4. Run the app

```bash
pnpm dev
```

Or use the startup script which handles everything automatically:

```bash
bash startup.sh
```

The app runs on [http://localhost:3000](http://localhost:3000). The Socket.IO WebSocket server runs on the same port.

## Demo Mode vs Non-Demo Mode

The app supports a **demo mode** for public-facing deployments where you want users to try the app without being able to make permanent changes.

### Enabling Demo Mode

Set both environment variables:

```env
DEMO_MODE=true
NEXT_PUBLIC_DEMO_MODE=true
```

### What Changes in Demo Mode

| Feature | Demo Mode | Non-Demo Mode |
|---------|-----------|---------------|
| **Login** | Redirects to a name picker (`/register`) — users enter a display name and get a temporary account | Standard email/password login or Google OAuth |
| **Registration** | Simplified name-only form | Full registration with email, password, name |
| **Messaging** | Enabled — users can send messages, react, and use all chat features | Same |
| **Voice/Video Calls** | Enabled — calls and huddles work normally | Same |
| **File Uploads** | Blocked — returns 403 | Enabled |
| **Create Workspace** | Hidden — the "Create Workspace" button is removed from the sidebar | Visible |
| **Create Channel** | Hidden — the "+" button next to Channels is removed from the sidebar | Visible |
| **Workspace Admin** | Destructive admin actions blocked | Full admin access |

### How It Works

- `DEMO_MODE` (server-side) — Controls API route guards via `demoBlock()` in `shared/lib/demo.ts`. Destructive mutations (workspace/channel creation, file uploads) return a `403 DEMO_MODE` error.
- `NEXT_PUBLIC_DEMO_MODE` (client-side) — Controls UI visibility. Hides buttons for creating workspaces and channels, simplifies the registration form to a name picker.
- The middleware redirects unauthenticated users to `/register` (name picker) instead of `/login` when demo mode is active.

│   │   ├── messages.ts       #   Message send/edit/delete/react
│   │   ├── channels.ts       #   Channel join/leave/create
│   │   ├── presence.ts       #   Online/offline heartbeat
│   │   ├── typing.ts         #   Typing indicators
│   │   ├── calls.ts          #   1:1 WebRTC call signaling
│   │   ├── huddles.ts        #   Group audio/video huddles
│   │   ├── polls.ts          #   Poll voting
│   │   ├── canvas.ts         #   Yjs document sync
│   │   └── read-receipts.ts  #   Read state tracking
│   ├── socket-auth.ts        # JWT auth for WebSocket connections
│   └── cron/                 # Scheduled message delivery
├── auth/                     # NextAuth v5 configuration
├── channels/                 # Channel CRUD, components, queries
├── messages/                 # Message CRUD, components, queries, store
├── calls/                    # Voice/video call components, hooks, store
├── canvas/                   # Collaborative editing (Tiptap + Yjs)
├── workflows/                # Automation engine
├── components/
│   ├── editor/               # Tiptap rich text editor + extensions
│   ├── layout/               # Sidebar, panels, navigation
│   ├── providers/            # Auth, Socket, Theme providers
│   └── ui/                   # 30+ Radix-based UI primitives
├── prisma/
│   ├── schema.prisma         # 22 models (SQLite dev / PostgreSQL prod)
│   └── seed.ts               # Demo data seeder
└── shared/

    └── types/                # TypeScript types (socket events, API)

## Tech Stack

- **Framework**: Next.js 14 (App Router, Server Components, Server Actions)
- **Database**: Prisma ORM with SQLite (dev) / PostgreSQL (prod)
- **Real-time**: Socket.IO (messaging, presence, typing, notifications)
- **WebRTC**: simple-peer (1:1 calls, huddles, screen sharing)
- **Rich Text**: Tiptap 3 with 11 extensions (mentions, code blocks, tables, etc.)
- **Collaboration**: Yjs (CRDT-based collaborative canvas editing)
- **Auth**: NextAuth v5 (JWT sessions, Google OAuth, credentials)
- **UI**: Radix UI + Tailwind CSS + Framer Motion
- **State**: Zustand (global app store)
- **File Storage**: Local filesystem (dev) / AWS S3 (prod) with sharp thumbnails
- **Testing**: Jest 30 + Testing Library (23 test suites)
